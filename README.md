# Objetivo da Atividade

Nesta atividade, vocês irão criar um protótipo de baixa fidelidade (Low-Fidelity Prototype) utilizando o Miro. O objetivo é praticar a representação rápida de interfaces, focando em estrutura, fluxo de navegação e organização de informação, sem se preocupar com cores, design visual refinado ou estética final. Prototipagem de baixa fidelidade é uma técnica amplamente utilizada em UX para validar ideias rapidamente, permitindo identificar problemas de usabilidade antes do desenvolvimento.

# Cenário do Problema

Uma startup chamada FilaZero deseja criar um aplicativo que ajude pessoas a evitar filas em estabelecimentos físicos (restaurantes, bancos, hospitais, órgãos públicos, etc.). O aplicativo permitirá que usuários:
• visualizem estabelecimentos próximos com filas
• vejam o tempo estimado de espera
• entrem virtualmente na fila
• recebam notificações quando sua vez estiver próxima
• avaliem a experiência no local
A empresa precisa de um protótipo inicial para apresentar a investidores.
